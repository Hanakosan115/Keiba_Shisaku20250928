   - `backtest_gui_logic_clean.py` 作成

### 主な発見

#### 🔴 重大問題1: データリーク

**Phase 13の219%回収率の正体**:
- MODE Bが current_date=None を使用
- predict_core()内でdatetime.now()が呼ばれる
- 2024年のレース予測時に2026年までの未来データにアクセス
- 結果: 非現実的な回収率（219%）

**真の回収率予想**: 80-100%程度

#### 🔴 重大問題2: 特徴量不一致

**症状**:
```
LightGBMError: The number of features in data (79) is not the same
as it was in training data (39).
```

**原因**:
- GUIが79個の特徴量を計算（Phase 12の古いロジック）
- Phase 13/14モデルは39個で訓練
- モデルと予測の不整合

**影響**:
- GUI予測が実行不可
- バックテストが実行不可
- Phase 2, Phase 3の実行がブロック

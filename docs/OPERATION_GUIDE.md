## データ品質の維持

### チェックリスト（週次）

- [ ] 未来データが混入していないか
- [ ] 重複データがないか
- [ ] 日付が連続しているか（抜けがないか）
- [ ] レース数が適切か（週末は多い、平日は少ない）

### 問題発生時の対処

**未来データが検出された場合:**
1. データ管理タブで「データクリーニング」を実行
2. 元データの取得設定を確認
3. `data_config.py` のパスを確認

**モデル訓練が失敗する場合:**
1. データ検証で品質スコアを確認
2. 訓練期間が適切か確認（最低1年以上推奨）
3. エラーログを確認

**予測が外れ続ける場合:**
1. バックテストで精度を確認
2. モデルタイプを変更してみる
3. 訓練データ期間を調整
4. 確信度フィルタを厳しくする（上位10%など）

## よくある質問（FAQ）

### Q1: データ更新の頻度は？
**A:** 週次推奨。毎週月曜に前週のレース結果を取得。

### Q2: モデル再訓練の頻度は？
**A:** 月次推奨。新しい月のデータが追加されたら再訓練。

### Q3: 回収率80%でも赤字では？
**A:** そうです。競馬は控除率25%なので、回収率100%未満は理論上赤字。80%台は「損失を最小化」レベル。90%以上を目指しましょう。

### Q4: 予測精度を上げるには？
**A:**
1. データ量を増やす（過去データをさらに取得）
2. 特徴量を追加（血統、調教データなど）
3. モデルのハイパーパラメータを調整
4. 確信度フィルタで厳選

### Q5: どの戦略がおすすめ？
**A:**
- **初心者**: ワイド_1-3（回収率87.7%、シンプル）
- **的中を楽しみたい**: 3連複_BOX5頭（的中率35.3%）
- **一発狙い**: カスタム戦略を自作

### Q6: race_idの調べ方は？
**A:** netkeibaのレースページURLの末尾12桁。
例: `https://race.netkeiba.com/race/result.html?race_id=202406030812`
→ `202406030812`

### Q7: 実際に馬券を買う際の注意点は？
**A:**
1. 予算を決める（月の上限を設定）
2. 確信度の高いレースのみ購入
3. 複数レースに分散投資
4. 記録を残して振り返る
5. **あくまで娯楽として楽しむ**

## 4. 提案・提言

### 提言 A — 予測確率のキャリブレーション補正（優先度: 高）

**現状の問題**:
Phase Aで判明した通り、モデルは全確率帯で**過小評価**している。
例：モデルが20〜25%と出す馬の実際の的中率は26.6%（+4.3pt）。

**提案**:
`Platt Scaling` または `Isotonic Regression` でキャリブレーションを補正し、
モデル出力をより正確な確率に変換する。

```python
from sklearn.calibration import CalibratedClassifierCV
# または
from sklearn.isotonic import IsotonicRegression

# キャリブレーション曲線のフィット（2024年検証データで）
iso = IsotonicRegression(out_of_bounds='clip')
iso.fit(raw_proba_val, target_val)

# GUI での使用
pred_calibrated = float(iso.predict([raw_proba])[0])
```

**期待効果**: Rule4 の閾値（pred_win > 20%）がより精確な馬を対象にできる。
キャリブレーション済みの確率でKelly計算をすると信頼性が上がる。

### 提言 B — Kelly基準によるベット金額の動的最適化（優先度: 中）

**現状**: 1点100円固定

**問題**: 期待値が高い馬（pred_win 30% × 20倍）と低い馬（pred_win 11% × 10倍）を
同じ金額でベットするのは非効率。

**提案**:
フラクショナルKelly（安全係数 0.25〜0.5）を適用する。

```python
# Kelly式: f = (p*b - (1-p)) / b
# p = 予測勝率（キャリブレーション後）
# b = 純利益倍率（オッズ - 1）
def kelly_bet(p, odds, fraction=0.25, unit=100, max_bet=500):
    b = odds - 1
    f = (p * b - (1 - p)) / b
    if f <= 0:
        return 0  # 期待値マイナス → ベットしない
    return min(max_bet, round(unit * f * fraction / unit) * unit)
```

**例**:
| pred_win | オッズ | Kelly(f) | ×0.25 | ベット額 |
|---:|---:|---:|---:|---:|
| 30% | 20x | 28.9% | 7.2% | 720円 |
| 20% | 10x | 11.1% | 2.8% | 280円 |
| 11% | 10x | 1.1% | 0.3% | 30円 → 見送り |

ペーパートレード期間中に **固定100円との比較を並行記録** しておくと、
将来の切り替えを判断できる。

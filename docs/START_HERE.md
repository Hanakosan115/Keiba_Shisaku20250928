### 1. GUIを起動

```bash

```


### 3. 予測実行

「予測実行」ボタンをクリック

**自動で以下が実行されます**:
- ✅ レース情報の取得（出馬表 or 結果ページ）

データ状態チェックで警告が出た場合:

```
[データ状態チェック]
  ℹ 7頭: データなし（予測精度低）
  ⚠ 2頭: データ古い（更新推奨）
```

→ 「🔄 このレースのデータ更新」ボタンをクリック

## 🔄 データ更新システム（3層アプローチ）

### レベル1: クイックチェック（自動）

**いつ**: 予測実行時に毎回自動
**処理時間**: <1秒
**動作**: 警告表示のみ

```
[データ状態チェック]
  ℹ 7頭: データなし（予測精度低）
  ⚠ 2頭: データ古い（更新推奨）
  ⚠ 1頭: 引退の可能性あり
    - マルシュロレーヌ (245日前)

  💡 ヒント: メニューから「データ更新」で最新データを取得できます
```

### レベル2: レース単位更新（手動）

**いつ**: 「🔄 このレースのデータ更新」ボタンクリック時
**処理時間**: 1-5分（馬数に依存）
**動作**: 出走馬の最新データを一括取得

```
[1] レース情報取得中... (ID: 202512280101)
  16頭の出走馬を確認

[2] 一括データ更新開始...

  更新が必要な馬: 7頭

  [1/7] オルフセン (ID: 2023105432)
    最終DB日付: 2024年10月15日
    → 3件の新規レースを発見

  [2/7] フォルテアンジェロ (ID: 2023106891)
    DB: データなし
    → 2件の新規レースを発見

  ...

更新完了!
  成功: 5頭
  新規なし: 2頭
```

### レベル3: 定期メンテナンス（スケジュール）

**いつ**: 週1回or月1回（自動実行推奨）
**処理時間**: 30分-2時間
**動作**: データベース全体の古いデータを更新

```bash
# コマンドライン実行
py smart_update_system.py periodic

# Windows タスクスケジューラに登録
トリガー: 毎週日曜日 02:00
動作: py C:\...\smart_update_system.py periodic
```

## 🛠️ トラブルシューティング

### Q: 「出馬表の取得に失敗しました」と出る

**A**: 以下を確認:
1. レースIDが正しいか（12桁の数字）
2. インターネット接続
3. netkeiba がメンテナンス中でないか

**解決**: 過去レースの場合、自動的に結果ページから取得します

### Q: 全ての馬の勝率が0.1%になる

**A**: 以下の可能性:
1. データベースに馬のデータがない → レベル2で更新
2. horse_id が正しく取得できていない

**解決**: 「🔄 このレースのデータ更新」で最新データ取得

### Q: オッズが0.0になる

**A**: 過去レースの出馬表ページはオッズ情報なし

**解決**: 自動的に結果ページから取得します（システムが自動処理）

### Q: 更新が遅い

**A**: netkeiba への負荷を避けるため、1馬につき1秒待機しています

**解決**: 正常な動作です。16頭のレースで約20-30秒

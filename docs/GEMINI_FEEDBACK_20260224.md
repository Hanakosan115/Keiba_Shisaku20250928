### フィードバック 1: ペーパートレードログへの「取得時刻」追加

#### 指摘内容

> `paper_trade_log.csv` に `odds_recorded_time`（オッズ確認時刻、例：発走5分前など）
> という列を追加することをお勧めします。Rule4はオッズに依存するため、
> 「いつ取得したオッズで判断したか」を記録しておかないと、後でバックテストの
> 結果と突き合わせる際の検証精度が落ちてしまいます。

#### なぜ重要か

Rule4の条件は `odds >= 2.0x` や `odds >= 10.0x` など**オッズ閾値**に依存している。
バックテストで使用した `単勝` 列は「確定オッズ（レース確定後）」であり、
実運用で「発走5分前オッズ」を使って判断した場合とは値がズレる可能性がある。

```
バックテスト: 確定オッズ 12.3x → Rule4 対象
実運用:       発走5分前 11.8x → Rule4 対象 ← 記録しないと検証不能
              確定オッズ 13.1x（市場変動）
```

この記録があることで、将来的に「どの時点のオッズで判断すべきか」の
精度検証が可能になる。

#### 対応内容

**`paper_trade_log.csv`** — ヘッダーに `odds_recorded_time` 列を追加:

```
date,race_id,race_name,horse_name,horse_id,pred_win,pred_place,
odds,odds_recorded_time,bet_rule,bet_amount,result,payout,pl,bankroll,memo
```

**`paper_trade_add.py`** — 入力プロンプトを追加:

```python
odds_time = input("  オッズ取得時刻 (例: 発走5分前 / 09:50): ").strip()
```

記録例:

| 列 | 値の例 |
|---|---|
| odds | 12.3 |
| odds_recorded_time | 発走5分前 |
| odds | 12.3 |
| odds_recorded_time | 09:50 |

### フィードバック 2: Phase D-1（オッズドリフト特徴量）のデータ取得に関する懸念

#### 指摘内容

> 朝イチオッズと直前オッズの差分（odds_drift）を特徴量にするアイデアは強力だが、
> 過去のレース（2020〜2025年）において、netkeiba等から「朝イチのオッズ」を
> 遡ってスクレイピングするのは仕様上困難なケースが多い
> （多くは最終確定オッズで上書きされるため）。
>
> 過去の時系列オッズデータが必要な場合は、JRA-VAN Data Lab. などの
> 公式データプロバイダーからの取得を検討するか、
> 「今週末から自前でリアルタイム収集して蓄積していく」という
> 割り切りが必要になるかもしれません。

#### 背景と整理

| 方法 | 過去データ | リアルタイム | コスト | 難易度 |
|---|---|---|---|---|
| netkeiba スクレイピング | ❌ 確定オッズのみ | △ 発走前なら可 | 無料 | 低 |
| JRA-VAN Data Lab. | ✅ 時系列あり | ✅ | 有料（月額） | 中 |
| 自前リアルタイム収集（今から） | ❌（過去は不可） | ✅ | 無料 | 低〜中 |

#### Phase D-1 の方針変更

**変更前の想定:**
> 過去2020〜2025年のオッズドリフトを遡って取得 → 特徴量として再訓練

**変更後の方針:**

```
短期（今すぐ）: 自前リアルタイム収集を開始
  → 毎週末、発走30分前・5分前のオッズをスクレイピングして蓄積
  → paper_trade_log.csv の odds_recorded_time と紐付け

中期（3〜6ヶ月後）: 蓄積データでPhase D-1 実装
  → 訓練データに過去3〜6ヶ月のオッズドリフト列を追加
  → モデル再訓練（Phase D-2 と同時）

長期（必要に応じて）: JRA-VAN Data Lab. 検討
  → 有料だが2020年以降の時系列オッズが取得可能
  → 大規模な特徴量拡充が必要になった時点で再評価
```

#### 自前収集スクリプト（Phase D-1 実装前に作成予定）

収集対象:
- 発走30分前の単勝オッズ（`odds_30min`）
- 発走5分前の単勝オッズ（`odds_5min`）
- 確定オッズ（`odds_final`）

収集タイミング: 毎週土日レース日（手動またはタスクスケジューラ）

保存先（案）: `data/odds_timeseries/odds_YYYYMMDD.csv`

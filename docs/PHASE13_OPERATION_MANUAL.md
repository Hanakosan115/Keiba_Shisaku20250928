### Step 2: レースIDを入力

**レースIDの取得方法**:
1. netkeiba.comで対象レースを開く
2. URLから12桁のレースIDを抽出
   - 例: `https://race.netkeiba.com/race/shutuba.html?race_id=202607050612`
   - レースID: `202607050612`

**入力**:
- GUIの「レースID」欄に12桁を入力
- 「予測」ボタンをクリック

---

### Step 3: 予測結果を確認

**表示される情報**:
| 馬番 | 馬名 | 勝率予測 | 印 | オッズ | バリュー | 回収率優秀 |
|:---:|:---|---:|:---:|---:|---:|:---:|
| 5 | ロードマイル | 0.187 | ◎ | 6.2 | 0.026 | **✓** |
| 3 | サンライズ | 0.165 | ○ | 8.1 | 0.042 | **✓** |
| 7 | キングダム | 0.142 | ▲ | 10.5 | 0.047 | ❌ |

**重要な列**:
- **勝率予測**: モデルが予測した勝率
- **回収率優秀**: 15-20%確率帯の馬は「✓」マーク

---

### Step 4: 購入対象馬を特定

**購入条件**:
1. ✅ 「回収率優秀」列に「✓」マーク
2. ✅ 勝率予測が0.15～0.20の範囲
3. ✅ オッズが実力に見合っている（バリュー > 0）

**例**:
```
馬番5 ロードマイル
  勝率予測: 0.187 (18.7%)
  オッズ: 6.2倍
  バリュー: 0.026
  → 購入対象 ✓
```

---

### Step 5: 購入金額を決定

#### 基本ルール: 固定額（初心者向け）
```
1レースあたり: 資金の1-2%
例: 資金10万円 → 1,000-2,000円/レース
```

#### 高度なルール: ケリー基準（上級者向け）
```
購入額 = 資金 × (予測勝率 × オッズ - 1) / (オッズ - 1)

例:
  資金: 10万円
  勝率予測: 0.187 (18.7%)
  オッズ: 6.2倍

  購入額 = 100,000 × (0.187 × 6.2 - 1) / (6.2 - 1)
         = 100,000 × 0.1594 / 5.2
         = 3,065円
```

**推奨**: ケリー基準の1/2 or 1/4を使用（リスク軽減）
```
購入額 = 3,065円 × 1/4 = 766円 → 800円に丸める
```

---

### Step 6: 購入実行

1. **競馬場またはオンライン**で単勝を購入
2. **馬番とオッズを再確認**
3. **購入額を確認**して決済

## 💰 資金管理ルール

### 原則1: 1レースあたりの投資上限

**初心者**: 資金の1%
**中級者**: 資金の2%
**上級者**: ケリー基準の1/4

**例**（資金10万円の場合）:
- 初心者: 1,000円/レース
- 中級者: 2,000円/レース
- 上級者: 状況により500-3,000円/レース

---

### 原則2: 連敗時のルール

**5連敗した場合**:
- 1日休む
- 資金管理を見直す
- 予測ロジックに問題がないか確認

**10連敗した場合**:
- 1週間休む
- システムの再検証
- モデルの性能確認

**根拠**: 143.9%回収率でも連敗は発生する
```
勝率17%の場合:
  5連敗確率: (1-0.17)^5 = 39.1% → 珍しくない
  10連敗確率: (1-0.17)^10 = 15.3% → 稀だが起こりうる
```

---

### 原則3: 資金の増減管理

**目標設定**:
- 月間回収率: 110-150%（長期的に143.9%に収束）
- 許容損失: 月間資金の10%まで

**資金調整**:
- **資金が2倍になった場合**: 利益の50%を出金、残りで継続
- **資金が半分になった場合**: 運用を一時停止、検証実施

## 📊 運用ログの記録

### テンプレート（CSV形式）

**ファイル名**: `phase13_operation_log.csv`

**ヘッダー**:
```csv
日付,レースID,レース名,馬番,馬名,勝率予測,オッズ,購入額,結果,払戻,収支,累積収支,回収率
```

**記入例**:
```csv
2026-02-21,202607050612,東京11R,5,ロードマイル,0.187,6.2,1000,1,6200,5200,5200,620.0
2026-02-21,202607060812,京都12R,3,サンライズ,0.165,8.1,1000,4,0,-1000,4200,525.0
2026-02-22,202607070905,中山9R,7,キングダム,0.175,7.5,1000,1,7500,6500,10700,623.3
```

**列の説明**:
- **結果**: 実際の着順（1なら勝ち）
- **払戻**: 的中時の払戻金額（不的中なら0）
- **収支**: 払戻 - 購入額
- **累積収支**: これまでの総収支
- **回収率**: (累積払戻 / 累積購入額) × 100

---

### 集計方法（Excel/Google Sheets）

**日次集計**:
```
=SUMIF(日付列, 今日, 収支列)
```

**月次集計**:
```
=SUMIFS(収支列, 日付列, ">="&月初日, 日付列, "<="&月末日)
```

**回収率**:
```
=SUM(払戻列) / SUM(購入額列) × 100
```

## ⚠️ 注意事項

### リスク管理

1. **過信しない**
   - 回収率143.9%は長期的な平均
   - 短期的には大きく変動する可能性

2. **分散投資**
   - 1レースに資金を集中しない
   - 複数のレースに分散

3. **感情のコントロール**
   - 連敗してもルールを守る
   - 連勝しても購入額を増やさない

---

### システム的な注意点

1. **オッズの変動**
   - GUIで表示されるオッズは取得時点のもの
   - 実際の購入時は変動している可能性
   - 大きく変動した場合は購入を見送る

2. **出馬取消・騎手変更**
   - 予測後に状況が変わった場合は無効
   - 最新情報を必ず確認

3. **モデルの更新**
   - Phase 13モデルは2020-2025年データで訓練
   - 定期的な再訓練が推奨（半年～1年ごと）

## 📊 Phase 1: データ管理（完了）

### 1-1. データ統計表示

**機能:**
- 総行数、日付範囲、レース数、馬数の表示
- 年別データ分布
- 未来データの自動検出

**使用例:**
```
=== データ統計 ===

総行数: 238,176
日付範囲: 2020-01-05 ～ 2024-12-28

ユニークレース数: 17,278
ユニーク馬数: 30,902

年別データ数:
  2020年: 48,234行
  2021年: 47,891行
  2022年: 48,512行
  2023年: 46,358行
  2024年: 47,181行

✓ 未来データなし（正常）
```

### 1-2. データクリーニング

**処理内容:**
1. 未来データの自動削除（今日より後の日付）
2. 重複レコードの削除（race_id + horse_id）
3. CSVとJSONの両方を処理
4. クリーン済みファイルの自動生成

**出力:**
- `*_cleaned.csv`: クリーンなCSVデータ
- `*_cleaned.json`: クリーンなJSONデータ

**実行ログ例:**
```

  重複削除: 1,234行
  クリーン後: 236,942行
  保存: netkeiba_data_2020_2024_clean_cleaned.csv

[2/2] JSONデータをクリーニング中...
  元データ: 20,461レース
  無効レース削除: 3,183レース
  クリーン後: 17,278レース
  保存: netkeiba_data_payouts_2020_2024_clean_cleaned.json

============================================================
クリーニング完了！
============================================================
```

### 1-3. データ検証

**7つの検証項目:**

| # | 検証項目 | 説明 |
|---|---------|------|
| 1 | 未来データチェック | 今日より後の日付を検出 |
| 2 | 重複チェック | race_id + horse_idの重複を検出 |
| 3 | 欠損値チェック | 必須カラム（race_id, horse_id, Rank, date）の欠損 |
| 4 | 日付形式チェック | 無効な日付形式を検出 |
| 5 | Rank範囲チェック | 1-18の範囲外を検出 |
| 6 | レース整合性チェック | 出走頭数8-18の範囲をチェック |
| 7 | データ品質スコア | 総合評価（0-100点） |

**品質スコア評価基準:**
- 🌟 90-100点: 優秀
- ✓ 70-89点: 良好
- ⚠️ 50-69点: 要改善
- ❌ 0-49点: 問題あり

**検証結果例:**
```
============================================================
検証サマリー
============================================================

総合評価: 🌟 優秀 (95/100)

✓ 問題は見つかりませんでした！

============================================================
```

## 🛠️ 技術仕様

### アーキテクチャ

**クラス構造:**
```
KeibaAnalysisTool
├── UI作成
│   ├── create_data_tab()         # データ管理タブ
│   ├── create_model_tab()        # モデル訓練タブ
│   ├── create_backtest_tab()     # バックテストタブ
│   ├── create_predict_tab()      # レース予測タブ
│   └── create_settings_tab()     # 設定タブ
│
├── データ処理
│   ├── update_data_stats()       # 統計更新
│   ├── clean_data()              # クリーニング
│   └── validate_data()           # 検証
│
├── モデル処理
│   ├── train_model()             # 訓練（フロント）
│   ├── _train_model_background() # 訓練（バック）
│   └── evaluate_model()          # 評価
│
├── バックテスト処理
│   ├── run_backtest()            # バックテスト（フロント）
│   ├── _run_backtest_background()# バックテスト（バック）
│   └── export_results()          # 結果エクスポート
│
└── 予測処理
    ├── predict_race()            # 予測（フロント）
    └── _predict_race_background()# 予測（バック）
```

### バックグラウンド実行

**重い処理はすべてバックグラウンドで実行:**
```python
thread = threading.Thread(target=self._heavy_task)
thread.daemon = True
thread.start()
```

**メリット:**
- UIがフリーズしない
- 進捗をリアルタイム表示可能
- ユーザー体験の向上

### データフロー

```
1. 設定ファイル (keiba_tool_config.json)
   ↓ パス情報

2. データファイル
   ├── netkeiba_data_2020_2024_clean.csv    (238,176行)
   └── netkeiba_data_payouts_2020_2024_clean.json (17,278レース)

3. モデルファイル
   └── lightgbm_model_*.pkl

4. 処理結果
   ├── クリーニング済みファイル (*_cleaned.*)
   ├── 訓練済みモデル (model_*.pkl)
   └── バックテスト結果 (エクスポート可能)
```

## 📝 使い方（クイックスタート）

### Step 1: 設定

1. **設定タブ**を開く
2. ファイルパスを設定（自動検出済み）
3. 「設定を保存」をクリック

### Step 2: データ確認

1. **データ管理タブ**を開く
2. 「データ統計を更新」をクリック
3. データの状態を確認
4. 必要に応じて「データ検証」

### Step 3: モデル訓練

1. **モデル訓練タブ**を開く
2. 訓練期間を設定（例: 2020/01 ～ 2023/12）
3. モデルタイプを選択
4. 「モデル訓練開始」をクリック
5. ログで進捗を確認

### Step 4: バックテスト

1. **バックテストタブ**を開く
2. テスト年を設定（例: 2024年）
3. テストする戦略をチェック
4. 「バックテスト実行」をクリック
5. 結果を確認

### Step 5: 予測

1. **レース予測タブ**を開く
2. race_idを入力（例: 202406030812）
3. 「予測実行」をクリック
4. 予測結果と推奨馬券を確認

## 主要な発見

### 1. データリーケージの重大性
**Phase 12までの問題**:
- 馬の全期間統計を使用 → 未来データが含まれる
- バックテスト的中率47.3% vs 実データ20.1%
- 約2倍の過大評価

**Phase 13での修正**:
```python
# Before (Phase 1-12)
horse_data = df[df['horse_id'] == horse_id]
stats = calculate_stats(horse_data)  # 全期間

# After (Phase 13)
horse_data = df[df['date'] <= cutoff_date]  # 過去のみ
horse_data = horse_data[horse_data['race_id'] != current_race_id]
stats = calculate_stats(horse_data)
```

### 2. Calibrationの重要性
**効果**:
- 確率予測の精度が飛躍的に向上
- 特に高確率帯（40%+）での過信を修正
- 15-20%確率帯の発見につながる

### 3. 15-20%確率帯の発見
**Phase 13の最重要発見**:
- 勝率15-20%の予測馬が最も回収率が高い
- 回収率: 143.9%（95%CI: 128.8%-160.1%）
- 平均オッズ: 5.5倍
- 的中率: 26.4%

**なぜ15-20%が最適か**:
- 本命馬（40%+）: オッズが低すぎる
- 大穴馬（<10%）: 的中率が低すぎる
- 15-20%: オッズと的中率のバランスが最良

### 4. 市場の効率性
**観察された現象**:
- 超本命馬: 過剰に人気（オッズが低すぎる）
- 中穴馬: 適正評価されていない（チャンス）
- 大穴馬: リスクに見合わない

# サーバー負荷対策の改善まとめ

## 実装日時
2025-12-10

## 改善内容

### 1. ランダム遅延機能
**Before:** 固定遅延（1秒、0.5秒）
**After:** ランダム遅延（範囲指定）

- レース結果取得: 2-4秒（ランダム）
- 馬情報取得: 2-5秒（ランダム）
- 血統情報取得: 1.5-3秒（ランダム）

**効果:** Bot検出回避、より自然なアクセスパターン

### 2. バッチクールダウン機能
10頭処理ごとに30-60秒の長時間休憩を実施

- 処理: 馬10頭
- 休憩: 30-60秒（ランダム）
- User-Agent変更: 休憩後に自動変更

**効果:** サーバー負荷の分散、長時間の連続アクセス回避

### 3. レース間クールダウン
各レース処理後に5-10秒の休憩

**効果:** レース単位での負荷分散

### 4. リトライロジック（指数バックオフ）
通信エラー時の自動リトライ機能

- 最大リトライ回数: 3回
- 待機時間: 2秒、4秒、8秒（指数的に増加）
- User-Agent変更: リトライ時に自動変更

**効果:** 一時的なエラーに対する耐性向上

### 5. 進捗保存・再開機能
`collection_progress.json` に進捗を自動保存

- 保存頻度: 10レースごと
- 保存内容: 処理済みレースID、処理済み馬数、タイムスタンプ
- 再開時: 自動的に進捗ファイル読み込み、未処理分のみ実行

**効果:** 中断・再開が可能、長時間実行の安全性向上

## 設定パラメータ

```python
# レート制限設定（調整可能）
DELAY_RACE_MIN = 2.0        # レース結果取得の最小遅延（秒）
DELAY_RACE_MAX = 4.0        # レース結果取得の最大遅延（秒）
DELAY_HORSE_MIN = 2.0       # 馬情報取得の最小遅延（秒）
DELAY_HORSE_MAX = 5.0       # 馬情報取得の最大遅延（秒）
DELAY_PEDIGREE_MIN = 1.5    # 血統情報取得の最小遅延（秒）
DELAY_PEDIGREE_MAX = 3.0    # 血統情報取得の最大遅延（秒）
BATCH_SIZE = 10             # 何頭ごとに長時間休憩するか
BATCH_COOLDOWN_MIN = 30     # バッチ休憩の最小時間（秒）
BATCH_COOLDOWN_MAX = 60     # バッチ休憩の最大時間（秒）
RACE_COOLDOWN_MIN = 5       # レース間の最小休憩（秒）
RACE_COOLDOWN_MAX = 10      # レース間の最大休憩（秒）
MAX_RETRIES = 3             # リトライ最大回数
RETRY_BACKOFF_BASE = 2      # リトライ時の待機時間ベース（秒）
```

## 推定処理時間

### 1レースあたり（16頭想定）
- レース結果: 2-4秒
- 馬16頭 × (血統1.5-3秒 + プロフィール2-5秒 + 過去戦績2-5秒) = 96-208秒
- バッチクールダウン（10頭ごと）: 30-60秒 × 1回 = 30-60秒
- レース間休憩: 5-10秒
- **合計: 約133-282秒（2.2-4.7分）/レース**

### 8,467レース全体（2025年全データ）
- 最小: 8,467レース × 2.2分 = **約18,627分（310時間 = 13日間）**
- 最大: 8,467レース × 4.7分 = **約39,795分（663時間 = 28日間）**

### 推奨: 段階的実行
1. **テスト実行**: 10レース（約22-47分）
2. **小規模実行**: 100レース（約3.7-7.8時間）
3. **本番実行**: 全8,467レース（分割推奨）

- 再実行すると `collection_progress.json` から自動再開
- 進捗ファイルを削除すれば最初からやり直し

## 注意事項

1. **長時間実行が必要**: 全データ収集には最低13日間必要
2. **PC起動維持**: ノートPCの場合はスリープ設定を無効化
3. **ネットワーク安定性**: 有線LAN推奨
4. **段階的実行推奨**: 一度に全件ではなく、100-500レースずつ実行
5. **エラー監視**: 初回は数時間様子を見て、エラーが頻発しないか確認

## エラー時の対応

### 頻繁にブロックされる場合
- 遅延時間を増やす（例: DELAY_HORSE_MAX = 8.0）
- バッチサイズを減らす（例: BATCH_SIZE = 5）
- クールダウン時間を増やす（例: BATCH_COOLDOWN_MAX = 120）

### 進捗ファイルが壊れた場合
```bash
# 進捗ファイルを削除して最初から
del collection_progress.json
```

## Rule4 バックテスト（単勝 1点100円）

### Rule4 条件定義

```
条件A: pred_win ≥ 20% AND 2.0倍 ≤ odds < 10.0倍
条件B: pred_win ≥ 10% AND odds ≥ 10.0倍
Rule4 = 条件A ∪ 条件B（どちらか一方でも満たせばベット）
```

## 旧モデル（Phase 12）との比較

| 指標 | Phase 14（今回） | 旧Phase 12（モード A） | 差異の解釈 |
|------|-----------------|----------------------|------------|
| ◎的中率（2024） | **35.0%** | 58.8% | Phase 12はデータリーケージで過大評価 |
| ◎単勝全買い ROI | **140.9%** | 354.7% | 同上 |
| Rule4 ROI | **139.1%** | — | Phase 12はRule4評価なし |

### Phase 12が異常値を出していた理由

Phase 12モデル（79特徴量）は以下の問題があった：

1. **モデルの訓練データリーケージ**: 全期間データで統計計算した特徴量を使用
2. **累積統計の漏れ**: `total_win_rate` 等が「将来も含めた全成績」を参照
3. **バックテスト自体もMODE Bで動作**: `datetime.now()` で将来データを参照

Phase 14 + `run_gui_backtest.py` では `current_date = レース日付` を強制しているため、この問題は解消。

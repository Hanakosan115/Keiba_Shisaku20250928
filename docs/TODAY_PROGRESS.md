### 5. 新モデル訓練 ✅

**訓練スクリプト:**
- `train_with_class_info.py` 作成・実行

**新特徴量（27次元 → 32次元）:**
```python
従来の27次元 +
  race_class_rank    # クラスランク (0-7)
  is_promotion       # 昇級初戦フラグ ← ユーザー指摘の問題を解決
  is_demotion        # 降級フラグ
  is_debut           # デビュー戦フラグ
  class_rank_diff    # クラス差分
```

**訓練結果:**
```
訓練データ: 378,930サンプル（13,824レース、2020-2023年）
検証RMSE: 3.34（良好）
Early stopping: 243イテレーション
モデル保存: lightgbm_model_with_class_info.pkl
```

**特徴量重要度 TOP10:**
```
1. ninki（人気）         : 14,204,201 ← 圧倒的！
2. avg_rank（平均着順）  :  1,074,656
3. jockey_top3_rate      :    221,073
4. distance              :    203,381
5. escape_rate           :    202,184
6. avg_agari             :    177,185
7. min_rank              :    166,337
8. std_rank              :    161,082
9. age                   :    143,412
10. jockey_win_rate      :    123,582
```

**クラス関連特徴量の重要度:**
```
race_class_rank  : 54,052 (21位/32) ← 中程度の重要度
is_promotion     :  1,054 (29位/32) ← 低い
is_demotion      :      0 (31位/32) ← 使われず
is_debut         :      0 (32位/32) ← 使われず
class_rank_diff  :  1,732 (27位/32) ← 低い
```

**考察:**
昇級フラグの重要度が低い理由:
1. 昇級馬のサンプルが少ない（5.3%）
2. 人気（ninki）や平均着順で間接的に捉えられている
3. 昇級の影響自体が小さい（+0.05着）

ただし、特定のケース（例：未勝利→1勝）では効果があるはず。

## 💡 **重要な発見**

### 1. 人気（ninki）が最重要
- 特徴量重要度で圧倒的1位
- 市場の集合知は侮れない
- 穴馬狙いの難しさを示唆

### 2. クラス情報の重要度は中程度
- 予想より低い（21位/32）
- 既存特徴量で間接的に捉えられている可能性
- ただし、明示的に持つことで解釈性向上

### 3. 昇級の影響は小さい
- 昇級初戦でも成績はほぼ変わらず（+0.05着）
- 昇級できる馬は実力がある
- サンプルサイズの問題もある

### 4. データ鮮度は致命的
- 平均21日前、最大133日前
- 週次更新で改善必須

### 5. 現状では利益は出ない
- 最良でも回収率87.7%
- 市場の効率性は高い
- 大幅な改善が必要

## 📝 **教訓**

1. **特別戦のクラス判定は複雑**
   - 括弧内の表記を正しく解析する必要
   - 賞金額からの逆算が有効
   - マッピングテーブルの整備が重要

2. **特徴量の重要度は予測困難**
   - クラス情報は重要だと思ったが中程度
   - 人気が圧倒的に重要
   - データで検証することの重要性

3. **データ鮮度は最優先**
   - 古いデータでは正確な予測不可能
   - 自動化が必須

4. **市場の効率性は高い**
   - 簡単に利益は出ない
   - 継続的な改善が必要

## 発見された重大な問題

### 問題1: データリーク（MODE B）

**症状**: Phase 13の219%回収率という非現実的な結果

**原因**:
```python
# backtest_gui_logic.py MODE B
if MODE == 'B':
    current_date_param = None  # ← これが問題
```

**影響**:
- 2024年のレースを予測する際に、2026年までのデータを使用
- 未来データによる「後出しジャンケン」状態
- 回収率が異常に高くなる

**解決策**: MODE Bを完全削除

### 問題2: 特徴量計算の不一致

**症状**: GUIとモデルの特徴量数が異なる

**詳細**:
- **モデル期待**: 39特徴量
- **GUI計算**: 79特徴量（Phase 12の計算ロジック）
- **Phase 13特徴量リスト**: 39特徴量

**原因**: GUIが古いPhase 12の特徴量計算ロジックを使用

**影響**:
- GUI予測が実行できない
- バックテストとの一致検証が不可能
- Phase 13モデルが実質使用不能

**解決策**:
1. GUIの特徴量計算を39特徴量に修正
2. または、Phase 14で完全に新しいシステムを構築

## Phase 13 の真の問題

### 219%回収率の正体

**Phase 13で報告された結果**:
- 40-50%確率帯: 219.1%回収率
- 35-40%確率帯: 166.2%回収率
- 全10確率帯: 100%超

**真の原因**: MODE Bによるデータリーク

**証拠**:
```python
# 2024年1月1日のレース予測時
current_date = None  # MODE B
→ predict_core()内でdatetime.now() = 2026-02-21を使用
→ 2024-2026年の2年分の未来データにアクセス
→ 「後から答えを知った状態」で予測
→ 当然的中率が高い
```

**現実的な期待値**:
- 優秀なモデル: 80-100%回収率
- 極めて優秀: 100-120%回収率
- 150%超: ほぼ確実にリークあり

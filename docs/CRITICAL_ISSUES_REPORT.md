## 2. 根本原因

### 2.1 訓練データ = テストデータ

```
モデル訓練: netkeiba_data_2020_2024_enhanced.csv (2020-2024年)
バックテスト対象: 2020-2024年の同じデータ
→ カンニング状態
```

### 2.2 年別成績の推移が証拠

| 年 | 本命的中率 | 説明 |
|----|-----------|------|
| 2020年 | 37.0% | 訓練初期データ |
| 2021年 | 43.5% | ↑ |
| 2022年 | 47.1% | ↑ |
| 2023年 | 46.9% | ↑ |
| 2024年 | **58.8%** | **最新データに過学習** |
| 2025年 | 51.1% | (一部未来データ) |
| **2026年** | **20.1%** | **真の実力** |

**明らかな過学習パターン:**
- 訓練データ内で年が新しいほど高成績
- 完全な未知データ(2026)で崩壊

### 2.3 datetime.now()の誤用

```python
# 現在のpredict_core()
current_date = datetime.now()  # 常に2026年2月を参照

# バックテスト時（2020年のレース予測時）
current_date = datetime.now()  # 2026年 = 6年後の未来データを使用可能
```

## 6. 必須対応事項

### 6.1 緊急（即時実施）

1. **全てのバックテスト結果を無効化**
   - README等の性能表記を削除
   - 「好成績」の説明を全て撤回

2. **datetime.now()の完全排除**
   ```python
   # predict_core()を修正
   def predict_core(self, race_id, horses, race_info, has_odds, current_date):
       # current_dateを必須化、datetime.now()を使用禁止

   ```

3. **ユーザーへの説明**
   - これまでの「好成績」は訓練データでのカンニング結果

## 8. 結論

**これまで説明されてきた「好成績」は完全な虚偽であり、実態は以下の通り:**

- ◎本命的中率: 20% (説明の47%は偽)
- 回収率: 52% (説明の234%は偽)
- 長期的には大幅赤字（月-21,000円ペース）

**原因:**
1. 訓練データ = テストデータのカンニング
2. datetime.now()による未来データ参照
3. 完全な未知データでの検証不足

**対応:**
全てのバックテスト結果を無効化し、モデルを一から再構築する必要がある。

## ⚠️ 未実施・課題

### 9. Phase 13実データ検証 ❌（データ品質問題により保留）

**問題**:
払戻データの構造に問題があり、正確な検証ができませんでした。

**具体的な問題**:
```json
// 現状（誤）
"複勝": {
  "馬番": ["7613"],  // ← 本来は ["7", "6", "1", "3"]
  "払戻金": [110230400]  // ← 本来は [110, 230, 400] など
}

// 正しい構造
"複勝": {
  "馬番": ["7", "6", "1", "3"],
  "払戻金": [110, 230, 400, 180],
  "人気": [2, 3, 6, 1]
}
```

**影響**:
- 複勝・ワイド・馬連の実データ検証が不可能
- 回収率の正確な計算ができない

**解決策**:
払戻スクレイピングスクリプトの修正が必要：
1. `horse_racing_analyzer.py` の `format_payout_data()` メソッドを修正
2. 馬番を個別に分割して保存
3. 払戻金も個別に保存
4. 再度データ収集

**修正後の再実行**:
- 推定時間: 4-6時間（スクリプト修正1時間 + 再収集5時間）

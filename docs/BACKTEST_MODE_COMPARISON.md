## 結論と考察

### 1. 未来データは性能を「低下」させる

**予想に反する結果**:
- 未来データにアクセスできるモードBが的中率**38.6%**
- データを制限したモードAが的中率**47.3%**（+8.7pt）

**原因仮説**:
1. **ノイズの混入**: 未来の無関係なレース結果がノイズとして作用
2. **過学習の促進**: 過去データ計算時に未来情報が混入し、一般化能力低下
3. **特徴量の劣化**: 馬の過去成績集計に未来レースが含まれ、予測時点での真の実力を反映しない

### 2. 年別偏りはモデル訓練時期に起因

**両モードで同じ傾向**:
- 2020年: 低成績（37.1% vs 29.6%）
- 2024年: 高成績（58.8% vs 52.7%）

**原因**:
- Phase12モデルは2020-2024年のデータで訓練
- 訓練データに近い時期（2024-2025年）で高精度
- 初期の2020年は訓練データの最古部で代表性が低い可能性

**対策案**:
- 時系列分割での再訓練（例: 2020-2022訓練 → 2023-2024検証）
- 年度ごとのモデル更新（ローリング・ウィンドウ）
- 時期を特徴量に追加

### 3. predict_core()実装は正しく機能

**検証結果**:
- `current_date`パラメータによる日付フィルタが正常動作
- モードAとモードBで明確な差異
- リーケージ防止機能が意図通り実装されている

### 4. 実戦での推奨

**モードA（リーケージ防止版）を採用すべき理由**:
1. **的中率が高い**: 47.3% vs 38.6%
2. **予測精度が高い**: 勝率50%超で77.4%的中
3. **現実的**: 実際の予測時に得られるデータ範囲と一致

**モードB（GUI一致版）の問題**:
- 未来データが予測を歪める
- 過度に自信を持った予測（50%超が3,380件だが63.5%的中）
- 実戦では再現不可能

## 推奨アクション

### 1. GUI側のcurrent_date使用を検討

現在のGUIは `current_date=None` (datetime.now())を使用しているが、これは:
- バックテストより低い性能（38.6% vs 47.3%）
- 未来データによるノイズの影響

**提案**:
```python
# GUI予測時もレース日付を使用
current_date = race_info.get('date', '')  # レース日付
# または
current_date = datetime.now().strftime('%Y-%m-%d')  # 今日の日付
```

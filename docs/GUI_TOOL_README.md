# 競馬分析統合ツール v2.0

## 概要

データ管理からモデル訓練、バックテスト、予測まで一元管理できるGUIツールです。

## 新機能

### 🎯 改善ポイント

1. **未来データ自動防止**
   - 日付範囲の検証機能
   - リアルタイムで未来データを検出・警告

2. **統合インターフェース**
   - 5つのタブで全機能にアクセス
   - データ管理 → モデル訓練 → バックテスト → 予測の流れが直感的

3. **設定の永続化**
   - ファイルパスなどの設定を自動保存
   - 次回起動時に設定が復元される

## 各タブの機能

### 1. データ管理タブ

**機能:**
- データ統計の表示（行数、日付範囲、レース数など）
- 未来データの自動検出
- データクリーニング
- データ検証

**使い方:**
1. 「データ統計を更新」ボタンをクリック
2. 統計情報を確認
3. 未来データがあれば警告表示

### 2. モデル訓練タブ

**機能:**
- 訓練期間の指定（年月単位）
- モデルタイプの選択（3連単特化/汎用/複勝特化）
- 訓練ログのリアルタイム表示
- モデル評価

**パラメータ:**
- 開始: 2020年1月～
- 終了: 2023年12月まで（テスト用に2024年を残す）
- モデルタイプ: 3連単特化（デフォルト）

### 3. バックテストタブ

**機能:**
- テスト年の指定
- 複数戦略の同時テスト
- 確信度フィルタリング（上位10%/20%/30%）
- 結果のCSV出力

**テスト戦略:**
- ✓ ワイド_1-3（デフォルトON）
- ✓ 3連複_BOX5頭（デフォルトON）
- □ 馬連_1-2
- □ 3連単_1着固定

**確信度フィルタ:**
- なし: 全レースをテスト
- 上位10%: 最も確信度の高い10%のレースのみ
- 上位20%/30%: より広い範囲でテスト

### 4. レース予測タブ

**機能:**
- race_idを指定してレースを予測
- 推奨着順の表示
- 推奨馬券の提示

**使い方:**
1. race_id を入力（例: 202406030812）
2. 「予測実行」をクリック
3. 予測結果を確認

### 5. 設定タブ

**機能:**
- ファイルパスの設定
- クリーンCSVパスの指定
- クリーンJSONパスの指定
- モデルファイルパスの指定
- システム情報の表示

**設定方法:**
1. 各パスの「参照」ボタンでファイルを選択
2. 「設定を保存」をクリック
3. 設定はkeiba_tool_config.jsonに保存される

## 設定ファイル

`keiba_tool_config.json` に以下の設定が保存されます：

```json
{
  "data_dir": "作業ディレクトリ",
  "clean_csv": "クリーンCSVのパス",
  "clean_json": "クリーンJSONのパス",
  "model_file": "モデルファイルのパス"
}
```

## トラブルシューティング

### GUIが起動しない
```bash
pip install tkinter
```

### データが読み込めない
- 設定タブでパスが正しいか確認
- ファイルが存在するか確認

### エラーが発生する
- ステータスバー（下部）にエラーメッセージが表示されます
- ログを確認してください

## サポート

問題が発生した場合は、以下を確認してください：
1. Python 3.7以上がインストールされているか
2. 必要なライブラリ（pandas, tkinter）がインストールされているか
3. データファイルのパスが正しいか

## ❌ 欠損している重要特徴量

### 【影響度：高】

#### 1. ⭐ クラス情報（未勝利/1勝/2勝/3勝/オープン）
**現状**: 存在しない
**影響**: **極めて大きい**
**理由**:
- 昇級初戦は馬の実力が不明（レベルアップできるか）
- 降級馬は格下相手で有利
- クラス跨ぎの比較ができない
- **ユーザー指摘の問題**: 先週勝ってクラスが上がった馬を識別できない

**例**:
```
馬A: 1勝クラスで3連勝 → 2勝クラスへ昇級
     ↓
今週: 2勝クラス初戦
     → モデルは「3連勝中の好調馬」と判断
     → 実際は格上初挑戦で苦戦
```

#### 2. ⭐ 休養期間（前走からの日数）
**現状**: 存在しない
**影響**: **極めて大きい**
**理由**:
- 休み明けは調整不足
- 中1週、中2週、中3週で能力発揮度が変わる
- 長期休養明けはパフォーマンス低下

**データ**:
```
休み明け（30日以上）: 勝率 約4%
中2週（14日）      : 勝率 約8%
連闘（7日以内）    : 勝率 約6%
```

#### 3. ⭐ 展開・ペース予想
**現状**: 脚質分布のみ（静的）
**影響**: **大きい**
**理由**:
- 逃げ馬が複数いる → ハイペース → 差し有利
- 逃げ馬がいない → スローペース → 先行有利
- 現状は馬の脚質だけで、展開の有利不利を予測できない

#### 4. ⭐ コース適性（過去の同一コース成績）
**現状**: 距離のみ
**影響**: **大きい**
**理由**:
- 同じ1800mでも、東京と京都は別物
- コース形状（直線長さ、コーナー数）で適性が変わる
- 右回り・左回りの得手不得手

**例**:
```
馬B: 東京1800m → 3戦2勝
     京都1800m → 5戦0勝
     ↓
現状モデルは区別できない
```

#### 5. 血統の芝ダート・距離適性
**現状**: father, mother_fatherの列はあるが未使用
**影響**: **中～大**
**理由**:
- デビュー戦や実績少ない馬の予測に有効
- 芝→ダート転向、距離延長などで血統が重要

## 🚨 データ鮮度問題（致命的）

### 現状の問題

チェックスクリプトの結果:
```
2024年10月20日のレース（サンプル10頭）:
  - 13頭/20頭: 最新レースが30日以上前
  - 平均: 21日前のデータ

具体例:
  馬A: 最新レース 133日前（4ヶ月前！）
  馬B: 最新レース  28日前（1ヶ月前）
  馬C: 最新レース  15日前（2週間前）
```

### 影響

**ユーザー指摘の問題が発生**:
```
実際:
  先週（11/17）: 馬Dが1勝クラスで勝利 → 2勝クラスへ昇級
  今週（11/24）: 馬Dが2勝クラスに出走

モデルの認識:
  最新データ: 11/3のレース（3週間前）
  → 昇級情報が反映されていない
  → 1勝クラスの強い馬として予測
  → 実際は格上初挑戦で苦戦
```

### なぜこれが起きるのか

1. **手動更新の限界**
   - 毎週手動でスクレイピング → 忘れる・忙しい
   - 全レース更新に時間がかかる

2. **過去データ参照の問題**
   - 過去10走を参照 → でも最新10走が古い
   - 直近1ヶ月の変化を捉えられない

3. **リアルタイム情報の欠如**
   - クラス変動は即座に反映されない
   - 調子の上昇・下降を見逃す

## 💡 改善提案：自動週次データ収集システム

### アーキテクチャ

```
┌─────────────────────────────────────────┐
│  自動スケジューラ（Windows Task Scheduler） │
│  毎週月曜 朝7:00実行                       │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  data_collector.py                      │
│  ①先週のレース結果を取得                   │
│  ②出走馬の最新情報を更新                   │
│  ③クラス変動を検出                        │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  netkeiba_data_2020_2024_clean.csv      │
│  ← 新しいレコードを追加                    │
│  ← 既存馬のデータを更新                    │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  feature_updater.py                     │
│  ①騎手・調教師の直近成績を再計算           │
│  ②クラス変動フラグを追加                  │
│  ③休養期間を計算                         │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  model_retrainer.py（月次）             │
│  最新データでモデルを再訓練                 │
└─────────────────────────────────────────┘
```

### 実装ステップ

#### Phase 1: 週次データ収集（優先度：高）
```python
# auto_data_collector.py

1. 先週の全レース結果を取得
   - netkeibaから前週土日のレース一覧
   - 各レースの結果（着順、タイム、馬体重など）

2. 出走馬の最新情報を更新
   - 各馬の過去成績を更新
   - クラス変動を検出（昇級・降級）

3. データベースに追加
   - 新レコードをCSVに追加
   - 重複チェック

4. ログ記録
   - 何件追加したか
   - エラーがあったか
```

#### Phase 2: 特徴量の動的計算（優先度：高）
```python
# dynamic_feature_calculator.py

予測時にリアルタイム計算:
1. 休養期間 ← 前走日付から自動計算
2. クラス情報 ← レース名から抽出
3. 直近成績（2週間、1ヶ月） ← 最新データから計算
4. コース適性 ← 過去の同一コース成績
```

#### Phase 3: 新特徴量の追加（優先度：中）
```python
# advanced_features.py

1. 展開予想指数
   - レース内の脚質分布から
   - 逃げ馬の数、ペース予想

2. 枠順バイアス
   - コース別統計をテーブル化

3. 血統適性
   - 父・母父の芝ダート成績
   - 距離別成績

4. オッズ変動
   - 前日オッズも取得して変化率を計算
```

### 自動化スケジュール

```
毎週月曜 07:00: 先週のレース結果収集
毎週火曜 07:00: 特徴量再計算
毎月1日  03:00: モデル再訓練
毎日    12:00: 今週末のレース情報取得
毎日    09:00: オッズ情報更新（レース前日から）
```
